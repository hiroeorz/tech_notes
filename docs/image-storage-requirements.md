# Hiroe Tech Notes 画像保存・記事画像アップロード要件書

## 1. 概要

本ドキュメントは、Hiroe Tech Notes における画像保存先、記事本文への画像追加、管理画面での画像管理に関する要件を定義する。

画像保存はローカルファイルシステムではなく Cloudflare R2 を使用する。Rails 側では Active Storage を利用し、R2 は S3 互換ストレージとして接続する。

対象画像は以下とする。

- 記事本文中に挿入する画像
- OGP 画像
- プロフィール画像

ブログ背景は現状どおり CSS による背景表現を基本とし、本要件の画像アップロード対象には含めない。

## 2. 保存先要件

### 2.1 ストレージ

- 画像ファイルの保存先は Cloudflare R2 とする。
- Rails アプリケーションからは Active Storage の S3 互換サービスとして R2 に接続する。
- production 環境では R2 を Active Storage の標準保存先とする。
- test 環境では既存どおり Active Storage の `:test` service を使用し、外部サービスには接続しない。

### 2.2 配信方式

- 記事本文中の画像は public CDN 経由で配信する。
- Markdown に挿入する画像 URL は CDN の公開 URL とする。
- 挿入形式は以下とする。

```markdown
![代替テキスト](https://cdn.example.com/path/to/image.jpg)
```

- 実際の CDN ホスト名は環境変数または Rails credentials で設定できるようにする。
- R2 バケット名、アカウント ID、アクセスキー、シークレットキー、エンドポイント URL、公開 CDN ベース URL はコードに直接書かない。

### 2.3 既存画像の移行

- 現在ローカル Active Storage に保存されている画像の移行は実施しない。
- R2 切り替え後に必要な画像は再アップロードで対応する。

## 3. 記事画像アップロード要件

### 3.1 記事との紐づけ

- 記事本文中に追加する画像は、対象記事に紐づく添付画像として管理する。
- Rails モデル上は `Post` に対して複数画像を添付できる構成とする。
- 実装候補は `Post has_many_attached :images` とする。

### 3.2 新規記事作成時の扱い

- 画像アップロード前に記事が保存済みであることを必須とする。
- 新規記事で画像を追加する場合は、先に下書き記事を保存する。
- 未保存の記事作成画面では、画像アップロード操作を無効化するか、「下書き保存後に画像を追加できる」ことが分かる UI にする。

### 3.3 アップロード操作

- 管理者は記事編集画面で画像選択フォームから画像を選択できる。
- 画像選択後、フォーム全体の保存を待たずに非同期で画像をアップロードする。
- アップロード成功時、編集エリアの現在のカーソル位置に Markdown 画像記法を自動挿入する。
- Markdown の代替テキストは、初期値としてファイル名から拡張子を除いた文字列を使用する。
- 代替テキストを入力できる UI を設ける場合は、その入力値を Markdown に反映する。

### 3.4 挿入される Markdown

- アップロード成功時に挿入される Markdown は以下の形式とする。

```markdown
![代替テキスト](https://cdn.example.com/path/to/uploaded-image.jpg)
```

- URL は Rails の Active Storage redirect URL ではなく、public CDN の画像 URL とする。
- CDN URL はサーバー側で生成し、クライアントには生成済み Markdown または生成済み URL を返す。

### 3.5 プレビュー

- 記事編集画面のプレビューでは、挿入済み Markdown 画像が公開表示と同じ見た目で表示される。
- 既存の `MarkdownRenderer` を通して画像をレンダリングする。
- Markdown レンダリング時は現在の画像装飾仕様を維持し、画像を `figure.article-image` として表示する。

## 4. 画像制限

### 4.1 ファイル形式

記事画像としてアップロード可能な MIME type は以下を基本とする。

- `image/jpeg`
- `image/png`
- `image/webp`
- `image/gif`

OGP 画像およびプロフィール画像の許可形式を変更する場合は、記事画像と同じポリシーに揃えるか、用途別に差を明記する。

### 4.2 ファイルサイズ

- 記事画像の上限サイズは 10MB とする。
- 上限を超える画像は保存せず、管理画面にエラーメッセージを表示する。
- OGP 画像およびプロフィール画像の上限を変更する場合は、別途用途別の上限を定義する。

### 4.3 バリデーション

- サーバー側で MIME type とファイルサイズを検証する。
- クライアント側の `accept` 属性や事前チェックは補助扱いとし、サーバー側バリデーションを必須とする。
- 不正な画像がアップロードされた場合、Active Storage の添付を残さない。

## 5. 添付画像一覧・削除要件

### 5.1 添付画像一覧

- 記事編集画面に、その記事に紐づく添付画像一覧を表示する。
- 各画像について、少なくとも以下を表示する。
  - サムネイル
  - ファイル名
  - ファイルサイズ
  - CDN URL または Markdown 挿入用の情報
  - 削除操作
- 一覧から任意の画像を再度 Markdown に挿入できる操作を提供することが望ましい。

### 5.2 手動削除

- 記事本文から Markdown を削除しても、添付画像は自動削除しない。
- 管理者が添付画像一覧から明示的に削除した場合のみ、対象画像を削除する。
- 削除時は Active Storage の添付関係と R2 上の実ファイルを削除する。
- 削除前に確認 UI を表示する。

### 5.3 削除時の注意

- 削除対象の画像 URL が記事本文に残っている場合、公開記事上で画像切れが発生する。
- 初期実装では本文内参照チェックは必須としない。
- 将来的には、削除前に本文内で使用中かどうかを検出し、警告表示する機能を検討する。

## 6. OGP 画像・プロフィール画像

- OGP 画像とプロフィール画像は、既存どおり `SiteSetting` の Active Storage 添付として扱う。
- 保存先は記事画像と同じく R2 とする。
- 管理設定画面のアップロード UI は既存の操作性を維持する。
- 既存ローカル画像の移行は行わない。

## 7. セキュリティ・運用要件

- R2 の認証情報は Rails credentials または環境変数で管理し、リポジトリにコミットしない。
- public CDN で配信するため、アップロードされた記事画像は公開可能なものとして扱う。
- 管理者のみが画像をアップロード・削除できる。
- アップロードエンドポイントは管理者認証配下に置く。
- ファイル名はそのまま保存キーに使わず、Active Storage またはサーバー側で衝突しないキーを生成する。
- CDN URL 生成時に、ユーザー入力をそのままパスとして連結しない。

## 8. 実装方針

### 8.1 Active Storage 設定

- `config/storage.yml` に R2 用 service を追加する。
- R2 は S3 互換 API として設定する。
- production の `config.active_storage.service` は R2 service を参照する。
- test の `config.active_storage.service` は `:test` のままとする。

### 8.2 管理画面エンドポイント

- 記事画像アップロード用の管理者向けエンドポイントを追加する。
- 実装候補は `Admin::PostImagesController#create` とする。
- 記事画像削除用の管理者向けエンドポイントを追加する。
- 実装候補は `Admin::PostImagesController#destroy` とする。

### 8.3 クライアント側挙動

- 記事編集画面の画像選択フォームでファイル選択を検知する。
- `FormData` を使って画像アップロードエンドポイントへ送信する。
- 成功レスポンスから Markdown 文字列を受け取り、本文 textarea の現在カーソル位置に挿入する。
- 挿入後、カーソル位置を Markdown の直後に移動し、本文の文字数・プレビュー状態を更新する。
- 失敗時は管理画面内にエラーを表示し、本文には何も挿入しない。

## 9. 非対象

- 既存ローカル画像の R2 への移行
- 記事本文から削除された未使用画像の自動削除
- 画像編集、トリミング、リサイズ UI
- 非公開画像や署名付き URL による閲覧制御
- ブログ背景画像のアップロード管理
