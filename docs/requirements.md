# Hiroe Tech Notes 要件書

## 1. 概要

Tech Notes は、インフラ、クラウド、SRE、自動化、プログラミングなどの技術的な学びや実験ログを記録・公開する個人ブログサイトのRailsアプリである。

公開サイトとしての記事閲覧体験と、管理者向けの記事作成・編集・設定管理体験を提供する。画面デザインは `docs/images` 以下の画面イメージを基準とし、画像内に表現されている機能は原則として実現する。また、このデザイン性を忠実に再現するものとする。

## 2. 対象画面

- トップページ: `docs/images/top.png`
- 記事詳細ページ: `docs/images/show.png`
- 管理記事一覧ページ: `docs/images/list.png`
- 記事作成・編集ページ: `docs/images/edit.png`
- 管理者ログインページ: `docs/images/login.png`
- 管理設定ページ: `docs/images/settings.png`

## 3. 共通要件

### 3.1 デザイン方針

- サイト全体は「技術ノート」「手書きの実験メモ」を連想させる紙のノート風デザインとする。
- 背景には薄いアイボリー系の紙質感と方眼罫を使用する。
- 画面左端にはリングノートの穴またはリング表現を配置する。
- アクセントカラーはオレンジとし、以下に使用する。
  - アクティブなナビゲーション
  - 見出しの下線または左罫線
  - 主要ボタン
  - 強調ラベル
  - 重要な注意枠
- カード、フォーム、テーブルは薄い枠線と控えめな角丸で表現する。
- UIは過度に装飾せず、技術記事を読みやすく、管理操作を行いやすい密度にする。

### 3.2 共通ヘッダー

- 全画面上部に共通ヘッダーを表示する。
- 左側にサイト名 `Hiroe Tech Notes` を表示する。
- サイト名の下にサブコピー `技術を、実践し、言語化する。` を表示する。
- 公開側および管理側の通常画面では、以下のグローバルナビゲーションを表示する。
  - ホーム
  - 記事一覧
  - 実験ログ
  - タグ一覧
  - プロフィール
  - About
- 現在表示中のナビゲーション項目にはオレンジの下線を表示する。
- 右側に以下のアイコン操作を表示する。
  - テーマ切り替え
  - 検索
- ログイン画面では、グローバルナビゲーションは表示せず、サイト名、サブコピー、テーマ切り替え、検索のみを表示する。

### 3.3 レイアウト

- 公開側のトップページと記事詳細ページは、本文エリアと右サイドバーの2カラム構成を基本とする。
- 管理側の一覧、編集、設定画面は、広い作業領域を確保するため全幅寄りの1カラム構成とする。
- 主要コンテンツはページ左右に適切な余白を設け、画面幅が広い場合も読みづらくならない最大幅を持たせる。
- スマートフォンや狭い画面では、2カラム構成を縦積みに切り替える。

### 3.4 アイコン

- 検索、テーマ、外部リンク、保存、プレビュー、追加、削除、鍵、画像、目、RSSなど、画面イメージにある操作にはアイコンを併用する。
- アイコンのみで意味が伝わりにくい操作にはテキストラベルを併記する。

### 3.5 データモデル

#### 記事

記事は以下の情報を持つ。

- タイトル
- スラッグ
- 要約
- 本文
- カテゴリー
- タグ
- ステータス
  - 公開
  - 下書き
  - レビュー中
- 公開日
- 更新日
- 読了時間
- 著者

#### カテゴリー

カテゴリーは以下の情報を持つ。

- 名前
- 表示順
- 記事数
- アイコンまたは識別表示

画面イメージ上のカテゴリー例:

- インフラ
- AWS
- Azure
- 自動化
- プログラミング
- セキュリティ
- AI開発
- 運用
- ポエム

#### タグ

タグは記事に複数付与できる。

画面イメージ上のタグ例:

- Terraform
- IaC
- AWS
- S3
- DynamoDB
- GitHub Actions
- Linux
- 運用
- 分析
- ベストプラクティス

#### プロフィール

プロフィールは以下の情報を持つ。

- 表示名
- 肩書き
- 自己紹介
- プロフィール画像
- メールアドレス
- SNS / 外部リンク
  - GitHub
  - X
  - RSS
  - Zenn
  - note

#### サイト設定

サイト設定は以下の情報を持つ。

- ブログタイトル
- サブタイトル / キャッチコピー
- サイトURL
- ブログ説明
- デフォルトOGP画像 / アイキャッチ
- デフォルトテーマ
- 記事一覧の表示件数
- プロフィール公開設定
- SNSリンク表示設定

### 3.6 認証・管理者機能

- 管理画面は管理者ログイン後のみ利用できる。
- 未ログイン状態で管理画面へアクセスした場合はログイン画面へ誘導する。
- ログインにはメールアドレスとパスワードを使用する。
- ログイン状態を保持するチェックボックスを提供する。
- パスワード再設定導線を表示する。
- 管理画面から記事の作成、編集、削除、ステータス変更、設定変更を行える。

### 3.7 検索・絞り込み

- 記事タイトルによる検索ができる。
- カテゴリーによる絞り込みができる。
- ステータスによる絞り込みができる。
- 並び順を変更できる。
- 絞り込み条件をリセットできる。

### 3.8 テーマ切り替え

- ヘッダーのテーマ切り替えアイコンから表示テーマを切り替えられる。
- 初期実装ではライトテーマを必須とする。
- ダークテーマは将来拡張として実装可能な構造にする。

### 3.9 フッター

- 記事詳細ページの下部に著作権表記を表示する。
- 表記例: `© 2025 Hiroe Tech Notes. All rights reserved.`

## 4. トップページ要件

### 4.1 ヒーローエリア

- 左側に大きな手書き風コピーを表示する。
  - `考えたことを、手を動かして確かめる。`
- 補足文を表示する。
  - `インフラとコードのあいだで揺れながら、学んだことや試したことをノートに残しています。`
- 特徴を3項目表示する。
  - クラウド / インフラ / 自動化が中心
  - 小さな実験と、そこから得た学びを記録
  - 誰かのヒントになればうれしいです
- 右側に「今日のひとことログ」カードを表示する。
- ひとことログカードには以下を表示する。
  - 日付
  - タイトル
  - コードスニペット
  - メモ
  - 実験ログへのリンク
- カードにはクリップなどの手書きノート風装飾を表示する。

### 4.2 最新の記事

- `最新の記事` セクションを表示する。
- カテゴリー絞り込みタブを表示する。
  - すべて
  - インフラ
  - AWS
  - 自動化
  - プログラミング
  - セキュリティ
  - AI開発
- 最新記事を一覧表示する。
- 各記事には以下を表示する。
  - 日付
  - タイトル
  - タグ
- 記事一覧ページへのリンクを表示する。

### 4.3 実験ログ

- `実験ログ（ミニ実験と検証の記事）` セクションを表示する。
- 実験ログをカード形式で表示する。
- 各カードには以下を表示する。
  - 日付
  - タイトル
  - 概要
  - タグ
  - 所要時間
- 実験ログ一覧へのリンクを表示する。

### 4.4 右サイドバー

- プロフィールカードを表示する。
- カテゴリー一覧を表示する。
- アーカイブ一覧を表示する。

#### プロフィールカード

- プロフィール画像を表示する。
- 表示名 `Hiroe` を表示する。
- 肩書き `インフラエンジニア / プログラマ` を表示する。
- 自己紹介文を表示する。
- GitHub、X、RSSへのリンクアイコンを表示する。

#### カテゴリー一覧

- カテゴリー名と記事数を表示する。
- `すべてのタグを見る` リンクを表示する。

#### アーカイブ一覧

- 月別アーカイブと記事数を表示する。
- `すべてのアーカイブを見る` リンクを表示する。

## 5. 記事詳細ページ要件

### 5.1 パンくず

- ページ上部にパンくずを表示する。
- 表示例:
  - ホーム
  - 記事一覧
  - Terraform

### 5.2 記事ヘッダー

- メインカテゴリーまたはタグをラベル表示する。
- 記事タイトルを大きく表示する。
- 管理者がログイン中の場合、記事タイトルのすぐ下に `Edit` リンクを表示し、該当記事の編集画面へ遷移できるようにする。
- 未ログイン状態では `Edit` リンクを表示しない。
- メタ情報を表示する。
  - 公開日
  - 更新日
  - 読了時間
  - 著者
- タグをチップ形式で表示する。
- 要約文を表示する。

### 5.3 記事本文

- Markdown由来の記事本文を読みやすく表示する。
- 見出しには左側のオレンジ罫線を表示する。
- 箇条書き、番号付きリスト、コードブロック、注意枠、画像または図解を表示できる。
- コードブロックは黒背景の技術記事向けスタイルで表示する。
- 注意枠は薄いオレンジ背景と枠線で表示する。
- 記事内の構成図を表示できる。

### 5.4 右サイドバー

- プロフィールカードを表示する。
- 目次を表示する。
- 関連タグを表示する。
- 関連記事を表示する。
- シェアリンクを表示する。

#### 目次

- 記事内の見出しを一覧表示する。
- クリックすると該当見出しに移動できる。

#### 関連タグ

- 記事に紐づくタグをチップ形式で表示する。

#### 関連記事

- 関連記事のタイトルと日付を表示する。

#### シェア

- GitHub、X、RSSなどのアイコンリンクを表示する。

## 6. 管理記事一覧ページ要件

### 6.1 パンくずと見出し

- パンくずを表示する。
- ページ見出し `記事一覧` を表示する。
- 見出し下にオレンジの手書き風下線を表示する。
- 右上に `新規作成` ボタンを表示する。

### 6.2 ステータス集計

- 以下の集計カードを表示する。
  - 全記事数
  - 公開記事数
  - 下書き記事数
  - レビュー中記事数
- 各カードにはアイコンまたはステータス色を表示する。

### 6.3 検索・フィルター

- タイトル検索入力を表示する。
- カテゴリー選択を表示する。
- ステータス選択を表示する。
- 並び順選択を表示する。
- フィルターリセットボタンを表示する。

### 6.4 記事テーブル

- 記事をテーブル形式で表示する。
- テーブル列は以下とする。
  - タイトル
  - カテゴリー / タグ
  - ステータス
  - 更新日
  - 操作
- タイトル列には記事アイコン、タイトル、本文または要約の抜粋を表示する。
- カテゴリー / タグ列にはチップを表示する。
- ステータス列には色付きバッジを表示する。
  - 公開: 緑
  - 下書き: オレンジ
  - レビュー中: 青
- 更新日は日時まで表示する。
- 操作列には以下を表示する。
  - プレビュー
  - 編集
  - 削除
- 削除操作は誤操作防止の確認を行う。

### 6.5 ページネーション

- 一覧下部にページネーションを表示する。
- 前へ、ページ番号、次へを表示する。
- 現在ページと総件数を表示する。

## 7. 記事作成・編集ページ要件

### 7.1 基本構成

- パンくずを表示する。
- 上部に記事の保存・公開操作を配置する。
- 執筆者が内容の作成・編集に集中できるよう、メイン画面ではタイトルと本文の入力に特化する。
- カテゴリーやタグ、スラッグなどのメタ情報は、画面上の「公開」ボタンを押した際に表示される公開設定ダイアログ（モーダル）で設定・変更するフローとする。
- 作成画面と編集画面は同じフォーム構成を使用する。

### 7.2 上部操作

- 以下の操作ボタンを表示する。
  - 下書き保存
  - プレビュー
  - 公開（公開設定モーダルを開く）

### 7.3 メイン編集フォーム（執筆画面）

- タイトル入力を表示する。
- タイトルには文字数カウンターを表示する。
- Markdown本文を入力するメインエディタエリアを表示する。
- ※カテゴリー、タグ、スラッグ、要約、種別、ステータス選択はメイン画面から除外し、公開設定モーダルへ集約する。

### 7.4 Markdownエディタ

- Markdownタブとプレビュータブを表示する。
- Markdown本文を入力できる大きなエディタを表示する。
- 行番号を表示する。
- シンタックスハイライト風の表示を行う。
- エディタ上部に以下のツールバーを表示する。
  - 太字
  - 斜体
  - コード
  - リスト
  - 引用
  - 番号付きリスト
  - チェックボックス
  - 画像
  - テーブル
  - リンクまたは外部参照
  - プレビュー
- Markdownガイドへのリンクを表示する。
- エディタ下部に以下を表示する。
  - 行数
  - 単語数
  - 文字数

### 7.5 プレビュー

- プレビュータブでは、記事詳細ページに近い形式でMarkdown本文を確認できる。
- プレビューボタンから別画面またはモーダルで公開前の表示を確認できる。

### 7.6 公開設定モーダル（公開ダイアログ）

- 上部の「公開」ボタンをクリックした際に、公開設定モーダルを表示する。
- 執筆内容の確認と合わせて、以下のメタ情報を設定する。
  - スラッグ入力
  - カテゴリー選択
  - 種別選択（記事 / 実験ログ）
  - タグ入力（カンマ区切り入力およびチップ表示に対応）
  - 要約（エクサープト）入力（文字数カウンターを表示）
  - 公開ステータス / 公開日時選択
- モーダル内に最終確定用の「投稿を公開する」ボタンおよび下書き保存・キャンセル操作を配置する。

## 8. 管理者ログインページ要件

### 8.1 レイアウト

- 共通ヘッダーを表示する。ただしグローバルナビゲーションは表示しない。
- 画面中央にログインカードを表示する。
- ログインカード上部にノート、鉛筆、ターミナルを連想させるイラストを表示する。
- 背景は他画面と同じノート風背景とする。

### 8.2 ログインフォーム

- 見出し `ログイン` を表示する。
- 説明文 `記事の作成・編集を行うためにサインインしてください。` を表示する。
- メールアドレス入力を表示する。
- パスワード入力を表示する。
- パスワード入力には表示/非表示切り替えアイコンを表示する。
- パスワード表示/非表示切り替えアイコンはボタンとして動作し、クリックまたはキーボード操作で入力欄の表示状態を切り替えられること。
- パスワード表示中は入力欄をテキスト表示にし、再度操作すると伏せ字表示に戻すこと。
- パスワード表示/非表示切り替えアイコンは、現在の表示状態が支援技術に伝わる属性またはラベルを持つこと。
- `ログイン状態を保持する` チェックボックスを表示する。
- `パスワードを忘れた場合` リンクを表示する。
- オレンジの `ログイン` ボタンを表示する。
- カード下部に `管理者専用ページです` の注記を表示する。

### 8.3 バリデーション

- メールアドレス未入力時はエラーを表示する。
- パスワード未入力時はエラーを表示する。
- 認証失敗時はログインできなかったことを表示する。

### 9.6 パスワード変更

### 9.7 最終更新

- 設定画面下部に最終更新日時を表示する。

## 10. 非機能要件

### 10.1 レスポンシブ対応

- デスクトップ、タブレット、スマートフォンで表示崩れがないこと。
- 管理画面のテーブルは狭い画面で横スクロールまたはカード型表示に切り替える。
- フォーム入力、ボタン、チップの文字がはみ出さないこと。

### 10.2 アクセシビリティ

- 主要な操作ボタンはキーボードで操作できること。
- フォーム入力にはラベルを関連付けること。
- アイコンのみの操作にはアクセシブルな名前を付与すること。
- 色だけに依存せず、テキストやアイコンでも状態を判別できること。

### 10.3 パフォーマンス

- トップページと記事詳細ページは、初回表示が重くならないように画像と装飾を最適化する。
- OGP画像やプロフィール画像は適切なサイズに変換または制限する。

### 10.4 セキュリティ

- 管理画面は認証済みユーザーのみアクセス可能とする。
- 記事本文のMarkdown表示時はXSS対策を行う。
- 削除、保存、公開などの管理操作にはCSRF対策を行う。
- パスワードは平文で保存しない。

## 11. 初期実装の前提
